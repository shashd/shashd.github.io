Java并发编程之四:原子类

原子类简介

编程中，原子性意味着“一组操作要么全部都成功，要么全部失败”，不会出现部分成功部分失败的情况。而java.util.concurrent.atomic下的类，就是具有原子性的类，可以原子性地进行添加、递增、递减等操作。

原子类的作用和锁类似，是为了保证并发情况下线程的安全性。相比于锁，原子性有以下的优点：

  • 粒度更细：原子变量将竞争范围缩小到变量级别，通常下，锁的粒度都大于原子变量的粒度
  • 效率更高：因为原子类的底层利用了CAS操作，不会阻塞线程

Atomic*基本类型原子类

基本类型原子类包含AtomicInteger、AtomicLong、AtomicBoolean。如AtomicInteger是int类型的封装，提供原子性的访问和更新，并发场景下可以直接使用该基本原子类。

Array数组类型原子类

数组中的元素都保证原子性。

一共有三种类型：AtomicIntegerArray、AtomicLongArray、AtomicReferenceArray

Reference引用类型原子类

可以让一个对象保证原子性。除了AtomicReference之外，还有AtomicStampedReference、AtomicMarkableReference。AtomicStampedReference是AtomicReference的升级，还加上了时间戳，用于解决CAS的ABA问题。AtomicMarkableReference和AtomicReference类似，多了一个绑定的布尔值，可以用于表示该对象已删除等场景。

FieldUpdater原子更新器

如果之前已经定义了一个不具有原子性的变量，可以使用FieldUpdater对已经声明的变量继续宁升级，使得其拥有原子性。

另一个场景是，在只有少数情况才会使用原子性的情况下，可以合理使用原子更新器进行合理的升级，而不是全部的变量都设置为原子类，这样一来可以节约内存。
